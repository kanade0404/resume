## 現状のギャップ

### 1. 直近（2025〜2026年）の実績が未反映

detail.md のサマリーには「LLM/AI Agent活用を主戦場」とあるが、本文に具体的なプロジェクト記述がない。以下を追記する。

- **AI Coding Tool の全社的な普及活動**
  - Claude Code / Codex / Devin の選定・導入・布教
  - 事業部の Biz メンバー向け AI リテラシー向上の取り組み
  - プラグイン開発、業務改善相談の受け付け
  - 事業部向け Claude Code 利用計測基盤の開発
- **社内業務のエージェント自動化**（調査・集計・レポーティング等）
- **AI 駆動開発の実践実績**
  - 主役はリリース回数ではなく**開発ハーネスの設計**として記述する。レビュアー不在という体制制約の中で、テスト・検証・監視・AI レビューをどう組み合わせて品質を担保したかを説明する
  - 継続的なリリース頻度と障害ゼロは、ハーネスの有効性を示す検証結果として添える（対象期間・対象プロダクト・リリースの定義・障害ゼロの観測範囲を明記）
  - 所属先での実績と個人開発を分けて記述する

なお、detail.md の既存記述は「Claude Code / Cursor / Devin」、直近の活動は「Claude Code / Codex / Devin」とツール名が異なる。時期による使い分け（Cursor は過去利用）が分かる形で表記を整合させる。
