# リサーチ結果のまとめ方

調べた結果を他人（未来の自分含む）が使える形に変換する。読み手の時間を尊重するフォーマットを選ぶ。

## 王道の骨格

### IMRAD

論文の定型。調査レポートにも流用できる。IMRAD は Introduction, Methods, Results, (And) Discussion の頭文字で、章立てとしては4セクション。

- **I**ntroduction — なぜこの問いか、先行調査は
- **M**ethods — 何をどう調べたか（再現可能に）
- **R**esults — 何が見つかったか（解釈抜き）
- **D**iscussion — 何を意味するか、限界は、次は何か

重要: Results と Discussion を分ける。「事実の記述」と「解釈」を混ぜない。

### BLUF（Bottom Line Up Front）

軍事・ビジネス発祥。結論を冒頭に置く。

忙しい読者は TL;DR しか読まない前提で、**最初の3行で何が言いたいか**を言い切る。

### Pyramid Principle（Barbara Minto / McKinsey）

トップダウン構造:

```
     [要旨]
     /  |  \
  [根拠1][根拠2][根拠3]
    |     |     |
  [詳細] [詳細] [詳細]
```

- 結論 → 根拠 → 詳細
- 各層は MECE
- どの層で止めても意味が通る

### SCQA（Minto の導入定型）

- **S**ituation — 合意できる前提状況
- **C**omplication — 問題の発生
- **Q**uestion — 立ち上がる問い
- **A**nswer — 本稿の答え

エグゼクティブサマリーやイントロに有効。

## 意思決定支援の構造

### Decision Matrix

評価軸 × 候補のマトリクス。重み付けスコアで定量化できる。

| 評価軸 | 重み | 候補A | 候補B | 候補C |
|-------|------|-------|-------|-------|
| 学習曲線 | 0.2 | 4 | 3 | 5 |
| 性能 | 0.3 | 5 | 4 | 3 |
| エコシステム | 0.3 | 3 | 5 | 4 |
| 長期保守性 | 0.2 | 4 | 4 | 3 |
| **加重合計** | | **3.9** | **4.1** | **3.8** |

**注意**: 重みは恣意的になりがち。**感度分析**を添える（重みを動かして順位がどう変わるか）。重み変動で順位が入れ替わるなら、結論は頑健でない。

### Trade-off Analysis

「何を得る代わりに何を失うか」を明示。これを言わない推奨は不誠実。

関連技法:

- **Wardley Map**: 進化段階 × 価値連鎖でマッピング
- **Eisenhower Matrix**: 重要度 × 緊急度
- **Cost-Benefit Analysis**: 数値化できる範囲で

### ADR フォーマット（Michael Nygard）

```
# ADR-NNNN: タイトル
Status: Proposed | Accepted | Deprecated | Superseded by ADR-XXXX
Date: YYYY-MM-DD

## Context
意思決定が必要になった背景・制約・代替案

## Decision
何を決めたか（1-2文）

## Consequences
### Positive
### Negative
### Neutral（副作用・依存の変化）
```

Aegis プロジェクトでは **実装時に** 書く（CLAUDE.md）。

## 引用管理

- **Zotero**: OSS、ブラウザ拡張で1クリック保存
- **BibTeX / CSL-JSON**: 標準フォーマット
- **DOI / arXiv ID**: 恒久 URL。素の URL より堅牢
- **Wayback Machine / archive.today**: 消えるページを即アーカイブ。企業ブログは特に消える
- **Permalink**: GitHub のコードは `y` でコミット固定 URL に

レポートに貼るリンクは可能な限り Wayback 等でアーカイブも併記する（特に一次情報）。

## 可視化

### Tufte の原則

- **Data-ink ratio 最大化** — データを表す部分のインクを最大化、装飾は削る
- **Chartjunk 排除** — 意味のない装飾・3D効果・虹色グラデは捨てる
- **Small multiples** — 同じ形式のグラフを並べて比較

### テキストで書ける図

- **Mermaid** — シーケンス、フロー、ER、ガント
- **PlantUML** — UML 各種
- **D2** — 新世代、美しい

テキストで書ける図は diff が取れる。レポジトリで一緒に管理できる。

## 禁じ手

- **結論だけ書いて根拠を書かない** — 追試できない
- **根拠だけ書いて結論を書かない** — 読者が推測する余地を残すな
- **数字を出すが出典を書かない** — 元ソースに遡れない数字は使うな
- **一次情報を見ずに断言する** — 二次情報からの孫引きは特に危険
- **反論を書かない** — Steelman なしの推奨は信頼できない
- **「詳しくは別紙」で逃げる** — 要旨は要旨内で完結させる
